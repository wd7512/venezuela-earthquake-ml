Contribuir

Este centro es un recurso público. Si usted es investigador, científico de datos o ingeniero que puede ayudar a llenar las brechas a continuación, esta página explica dónde puede participar.

¿Busca donar o compartir información? Vea Cómo Ayudar — esta página es para desarrolladores y científicos de datos que quieran contribuir con datos, mapeo o análisis.

Gracias por querer contribuir. A continuación, las áreas donde sus habilidades pueden tener el mayor impacto. Las pistas de alta prioridad necesitan colaboradores con experiencia en ML, teledetección, análisis geoespacial y PLN.

Cómo Ayudar

1. Elija una Brecha

Visite la página de Brechas para una lista priorizada de dónde se necesita ayuda. Las brechas de alta severidad son los puntos de partida más impactantes.

2. Reclame o Abra una Vía

A continuación hay seis vías de contribución. Cada una tiene fuentes de datos, enfoques sugeridos e hitos. Si quiere trabajar en una, abra un issue o envíe un PR con su enfoque propuesto.

3. Ejecute la Compilación

uv sync
uv run python build_site.py
# Salida va a dist/

Pipeline de Datos

Los datos del sitio viven en data.json. Un trabajo cron lo regenera cada 6 horas desde INTEL_SCOUTING.md y fuentes en vivo. Para actualizar manualmente:

uv run python build_site.py    # regenera gráficos + reconstruye todas las páginas desde data.json

Repositorio

Los datos del sitio viven en data.json. El nombre del repositorio refleja los componentes de aprendizaje automático utilizados en el procesamiento de datos (evaluación de daños por satélite, estimación de víctimas). El sitio de cara al público no es un sistema ML. Fuente: github.com/wd7512/venezuela-earthquake-ml
