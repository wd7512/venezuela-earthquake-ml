Contributing

This hub is a public resource. If you're a researcher, data scientist, or engineer who can help fill the gaps below, this page explains where to plug in.

Looking to donate or share information? See How to Help — this page is for developers and data scientists who want to contribute data, mapping, or analysis.

Public-first: the pages visible in the main navigation (Overview, Intel, Handbook, How to Help, Sources, Statistics) are for responders and the public. This Contributing section is for people who want to contribute data, mapping, or analysis.

How to Help

1. Pick a Gap

Visit the Gaps page for a prioritized list of where help is needed. High-severity gaps are the most impactful starting points.

2. Claim or Open a Track

Below are six contribution tracks. Each has data sources, suggested approaches, and milestones. If you want to work on one, open an issue or submit a PR with your proposed approach.

3. Run the Build

uv sync
uv run python build_site.py
# Output goes to dist/

Data Pipeline

The site data lives in data.json. Rebuilt as needed from INTEL_SCOUTING.md and live sources. To update manually:

uv run python build_site.py    # regenerates charts + rebuilds all pages from data.json

Repository

The site data lives in data.json. The repository name reflects the machine-learning components used in data processing (satellite damage assessment, casualty estimation). The public-facing site is not an ML system. Source: github.com/wd7512/venezuela-earthquake-ml
